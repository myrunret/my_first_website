Under the hood

How this site is built

A small Flask app with static front-end assets — fast to host, easy to audit, and readable without a heavy JavaScript framework.

Purpose

This project combines narrative scrolling, a little data visualisation, and clear sourcing so visitors can move from awareness to action. The stack is intentionally boring: fewer moving parts means fewer surprises in deployment or coursework demos.

Data & privacy

Production numbers are served from a local SQLite database checked into the repo for reproducibility. The pledge on the action page is stored in your browser only (localStorage) unless you later wire it to a server.

A /track-section route exists for lightweight session-based engagement experiments; it does not replace a full analytics policy — say so in your own deployment if you enable it broadly.

Technology stack

Python · Flask
Server routes, Jinja templates, and JSON endpoints. SQLite via sqlite3 and Flask’s g for per-request connections.
Templates · Jinja2
base.html holds shared chrome; pages extend it and override {% raw %}{% block %}{% endraw %} sections (for example content and title).
Front-end
Semantic HTML, one stylesheet (styles.css), and vanilla JavaScript (script.js) for scroll progress, intersection observers, canvas charts, and the pledge UI.
Fonts
Playfair Display, DM Sans, and DM Mono loaded from Google Fonts (see CSS @import).
Tooling & AI
Editor and AI assistants may help with copy, refactors, and accessibility passes — human review remains responsible for accuracy and citations (see Data & sources).
